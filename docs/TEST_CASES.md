# 测试用例

one-key-prompt skill 的手动测试清单。每次修改后跑一遍。

## 正向炼金流程

### TC-01: 写作场景 — 完整四步
输入：
> 帮我写一篇产品文案，要让人觉得这个产品很特别，不是那种烂大街的推销感，要有一种好像朋友在跟你聊天的感觉，看完之后很想买但又不觉得被推销了，语气要轻松但不是随便，要让人觉得这个品牌很懂自己。

期望：
- 模糊度 6-9
- 效果预估 ≤ ★★★☆☆
- 炼金结果包含人物/理论/概念三个类别各至少 1 个推荐
- 每个推荐有「💡 为什么适合」和「📋 直接用」
- 每个类别有「🤔 也考虑了」备选
- 对比环节有原版字数和炼金版字数

### TC-02: 策略场景 — 理论推荐
输入：
> 帮我分析一下我们公司该不该进入新能源市场，现在竞争挺激烈的，但我们有渠道优势

期望：
- 炼金结果中理论类推荐应包含分析框架类（如波特五力、SWOT 等）
- 推荐理由引用了用户输入中的具体内容（如“渠道优势”）

输入：

期望：收到「内容太短了，请补充更多细节或粘贴你的完整提示词」提示，流程停止。

### TC-04: 精准输入
输入：

期望：模糊度 ≤ 3，收到「你的提示词已经很精炼了」反馈。

### TC-05: 超长输入
输入：粘贴一段 2000+ 字的内容

期望：收到截断提示或正常处理不报错。

## 输出格式自适应

### TC-11: 编程类任务
输入一段关于代码架构的提示词。

期望：推荐偏向方法论/设计模式/工程实践，语气偏技术化。

### TC-12: 写作类任务
输入一段关于文案写作的提示词。

期望：推荐偏向叙事手法/人物风格，语气偏感性。
