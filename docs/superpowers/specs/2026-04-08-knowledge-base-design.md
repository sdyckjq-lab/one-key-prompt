# 设计文档：v2 激活钥匙知识库

日期：2026-04-08
状态：APPROVED

## 背景

v1 版本的提示词炼金术完全依赖 AI 模型推荐激活钥匙。v2 引入精选知识库，让推荐从"AI 自由发挥"升级为"知识库兜底 + AI 灵活补充"的混合模式。

## 核心定位

**混合模式**：先查知识库找匹配条目，没命中再让 AI 自由发挥。知识库是建议，不是约束——命中的条目注入 Agent prompt 作为参考候选，但 AI 可以推荐知识库以外的概念。

## 知识库规模

初始 50-80 个精选条目，分布在三个类型文件中，每个文件 20-30 条。

## 文件结构

```
skill 内置：
  knowledge-base/
    people.json        # 人物类激活钥匙
    theories.json      # 理论/模型类激活钥匙
    concepts.json      # 概念/隐喻类激活钥匙

用户扩展（项目目录下）：
  .prompt-alchemy/
    knowledge-base/
      people.json      # 追加的人物条目
      theories.json    # 追加的理论条目
      concepts.json    # 追加的概念条目
```

### 合并规则

- skill 先读内置知识库，再读用户扩展
- 同 id 条目：用户版覆盖内置版
- 不同 id 条目：合并到候选池
- 用户扩展文件不存在 → 纯用内置，不影响运行

## 条目字段定义

每个 JSON 文件结构：

```json
{
  "version": 1,
  "entries": [
    {
      "id": "taleb",
      "name": "塔勒布",
      "name_en": "Nassim Nicholas Taleb",
      "core_idea": "反脆弱——从混乱和不确定性中获益的能力",
      "scenarios": ["分析", "策略", "风险管理"],
      "keywords": ["不确定性", "风险", "黑天鹅", "反脆弱"],
      "prompt_template": "用塔勒布的反脆弱框架分析{topic}",
      "popularity": "high",
      "related": ["antifragile", "black-swan"]
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 唯一标识，格式：`{英文名或缩写}` |
| `name` | string | 是 | 中文名称 |
| `name_en` | string | 否 | 英文名称，便于跨语言匹配 |
| `core_idea` | string | 是 | 一句话核心说明 |
| `scenarios` | string[] | 是 | 适用场景标签（一级场景，至少 1 个） |
| `keywords` | string[] | 是 | 触发关键词，用于匹配用户输入 |
| `prompt_template` | string | 是 | 提示词模板，`{变量}` 标记可替换部分 |
| `popularity` | enum | 是 | `high` / `medium` / `low` |
| `related` | string[] | 否 | 关联条目 id 列表 |

### keywords vs scenarios 的区别

- **keywords**：从用户提示词文本中匹配（如用户提到"风险"→ 命中塔勒布）
- **scenarios**：从诊断结果的任务类型匹配（如诊断结果是"策略分析"→ 命中波特五力）

## 场景标签体系

### 一级场景（必选，固定枚举）

| 场景 | 说明 | 典型任务 |
|------|------|---------|
| `写作` | 内容创作、文案、故事 | 写文章、写脚本、写广告 |
| `分析` | 数据分析、市场分析、问题诊断 | 竞品分析、用户调研、财务分析 |
| `策略` | 规划、决策、商业模式 | 制定战略、商业模式设计 |
| `创意` | 头脑风暴、设计、创新 | 起名、设计方案、创新思维 |
| `编程` | 代码、架构、技术方案 | 写代码、系统设计、调试 |
| `沟通` | 说服、演讲、谈判 | 写邮件、做汇报、谈判策略 |
| `学习` | 知识整理、教学、研究 | 读书笔记、概念解释、课程设计 |

### 二级标签（可选，可自由添加）

- 写作 → `说服型写作`, `叙事型写作`, `技术写作`
- 分析 → `定量分析`, `定性分析`, `根因分析`
- 策略 → `竞争策略`, `增长策略`, `风险策略`
- 创意 → `发散思维`, `收敛思维`, `跨界联想`
- 编程 → `架构设计`, `代码优化`, `调试排错`
- 沟通 → `说服`, `演讲`, `谈判`, `反馈`
- 学习 → `知识提炼`, `类比教学`, `费曼技巧`

## 匹配逻辑

### 查询流程

```
诊断结果
    ↓ 提取 keywords + scenario
查知识库（对应类型文件）
    ↓ 场景匹配 + 关键词交集
    ↓ score = keywords_overlap × 2 + scenario_match × 1
    ↓ 取 top 3
注入 Agent prompt 的「参考知识库」字段
    ↓
Agent 在知识库建议 + 自由发挥之间选择
```

### Agent prompt 改造

在 v1 Agent prompt 的推荐规则部分，可选插入：

```
参考知识库（优先从中选择，也可以推荐知识库以外的）：
- 塔勒布：反脆弱——从混乱中获益。适合场景：分析、策略。模板：用塔勒布的反脆弱框架分析{topic}
- 波特五力模型：分析行业竞争格局。适合场景：分析、策略。模板：用波特五力模型分析{topic}的竞争格局
...
```

### 设计要点

- 知识库是建议不是约束，Agent 可以忽略不合适的命中
- 简单关键词匹配，不做语义搜索（20-30 条遍历无性能开销）
- 每个 Agent 只读自己类型的文件，减少无关信息

## 质量保证

### 条目准入流程

1. **AI 生成初稿**：对候选概念按 JSON schema 生成完整条目
2. **人工审核三要素**：
   - `core_idea` 是否一句话精准概括
   - `prompt_template` 是否实际好用（在 Claude/GPT 中实测）
   - `keywords` 是否覆盖用户真实表达（用词而非学术词汇）
3. **淘汰标准**：
   - prompt_template 实测效果不明显 → 淘汰
   - 太冷门（popularity: low 且非独特精准）→ 淘汰
   - 和其他条目高度重叠 → 合并或淘汰

## 不做的事

- 不做语义向量搜索（条目少，不需要）
- 不做自动更新/在线同步（纯本地文件）
- 不做条目评分/反馈系统（保持简单）
- 不限定领域（知识库是通用精选，用户可按需扩展）
