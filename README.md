# one-key-prompt

> 一个名字，比一万字描述更管用。

把冗长的提示词炼成精准的「概念激活钥匙」——用一个恰当的人名、理论或概念，比长篇大论更有效地激活 AI 的知识网络。

## 它解决什么问题

大多数人写提示词越写越长，试图用详细描述覆盖所有细节。但冗长的描述是在「教」AI，而不是「激活」AI 已有的知识网络。

**核心洞察**：告诉 AI「用塔勒布的反脆弱框架」，比花一千字描述「你要考虑从混乱中获益的能力」更有效。

## 效果对比

### 案例一：写文案

**你的原版（118 字）：**

> 帮我写一篇产品文案，要让人觉得这个产品很特别，不是那种烂大街的推销感，要有一种好像朋友在跟你聊天的感觉，看完之后很想买但又不觉得被推销了，语气要轻松但不是随便，要让人觉得这个品牌很懂自己。

**炼金版（18 字）：**

> 用拉斯韦尔的「只对一个人说话」原则写产品文案。

118 字描述的感觉，一个名字就激活了。

### 案例二：劝人

**你的原版（102 字）：**

> 我同事最近做了个很蠢的决定但我不好直说，我想劝他改主意但他这人特别固执不爱听别人的意见，你帮我想个说法，语气要温和但得让他意识到问题，不能伤他面子但是又得点中要害，最好让他自己得出结论。

**炼金版（15 字）：**

> 用苏格拉底式提问法劝同事改主意。

100 字在纠结"怎么说"，15 字直接锁定了"用什么方法说"。

### 更多场景

| 你写的 | 炼金后的钥匙 | 字数变化 |
|--------|-------------|---------|
| 帮我分析用户注册要验证邮箱密码并发发并发回滚第三方登录... | 用 ACID 事务 + 乐观锁设计注册流程 | 143→22 |
| 帮我想想我们公司竞争环境供应商议价新进入者替代品... | 用波特五力模型分析竞争格局 | 155→15 |
| 帮我复盘这个项目哪里做错了下次怎么避免... | 用 PDCA 循环做项目复盘 | 89→11 |

## 安装

```bash
# Claude Code 用户
claude mcp add-plugin https://github.com/sdyckjq-lab/one-key-prompt
```

安装后直接在对话中使用 `/one-key-prompt` 或 `/提示词炼金术` 即可。

## 使用方法

### 基本用法

直接调用命令，贴上你的提示词：

```
/one-key-prompt
```

然后输入你想优化的提示词。Skill 会自动完成三步：

**第一步：诊断** — 分析你的提示词哪里写得模糊、哪里可以精炼

```
🔍 诊断结果

模糊度：8/10
核心意图：写一篇有说服力但不像推销的产品文案
主要问题：
  - 用 100 多字描述一种感觉，而不是给出方法
  - "朋友聊天的感觉""不觉得被推销"→ 这些是结果，不是指令
```

**第二步：炼金** — 从三个维度推荐激活钥匙（人物 / 理论 / 概念）

```
✨ 炼金结果

👤 拉斯韦尔 — 「只对一个人说话」原则
📊 AIDA 模型 — 注意-兴趣-欲望-行动
🔑 知识诅咒 — 你忘了不懂产品的人是什么感觉
```

每个钥匙都附带一句可以直接复制使用的提示词。

**第三步：对比** — 原版 vs 炼金版，一目了然

### 深度探索

对某个激活钥匙感兴趣？输入「解读 + 名称」深入理解：

```
解读 苏格拉底式提问法
```

Skill 会解释这个概念的内涵、核心要点、在提示词中的高级用法，以及相关的激活钥匙。

### 反向炼金

已经知道一个概念名，想知道它适合什么场景？直接输入概念名加疑问词：

```
费曼学习法怎么用
```

Skill 会告诉你这个概念的内涵、适合和不适合的场景、以及可以直接使用的提示词模板。

## 知识库

内置 75 个精选激活钥匙，覆盖：

| 类型 | 数量 | 示例 |
|------|------|------|
| 人物 | 25 | 塔勒布、芒格、费曼、孙子、乔布斯... |
| 理论 | 25 | 波特五力、第一性原理、金字塔原理、蓝海战略... |
| 概念 | 25 | 确认偏误、橡皮鸭调试、复利效应、逆向思维... |

> 部分高人气条目附带「组合配方」，在解读模式下可以看到概念组合的推荐。

### 自定义扩展

在项目目录下创建 `.prompt-alchemy/knowledge-base/` 文件夹，添加自己的激活钥匙：

```json
{
  "version": 1,
  "entries": [
    {
      "id": "differential-diagnosis",
      "name": "鉴别诊断思维",
      "core_idea": "系统排除法，从症状推导最可能的病因",
      "scenarios": ["分析", "学习"],
      "keywords": ["诊断", "排查", "鉴别", "排除法", "病因分析"],
      "prompt_template": "用鉴别诊断的思维框架分析{topic}，列出可能的原因并逐一排查",
      "popularity": "medium"
    }
  ]
}
```

详见 `.prompt-alchemy/knowledge-base/README.md`。

## 原理

灵感来自两个认知科学洞察：

1. **隐性知识理论**（Michael Polanyi）—— 我们知道的东西远多于能说出来的。AI 模型也是：它「知道」的东西远多于你能在提示词里描述的。精准的概念名称能激活这些隐性知识。

2. **参数空间的知识组织**（Ilya Sutskever）—— AI 模型的参数空间中，人名和概念标签对应着高度组织化的知识簇。提到「费曼」，AI 就自动激活了费曼学习法、科学直觉、反直觉思维等一整套知识网络。
